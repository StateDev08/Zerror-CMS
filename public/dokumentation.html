ZerroCMS – Server-Dokumentation

Deployment-Anleitungen für alle gängigen Webserver. Document Root immer auf public/ setzen.

📋 Voraussetzungen

  • PHP 8.2 oder höher (mbstring, openssl, pdo_mysql, tokenizer, xml, ctype, json, bcmath, fileinfo)
  • MySQL oder MariaDB
  • Document Root des Webservers zeigt auf den Ordner public (nicht auf das Projekt-Root)

🔶 Apache .htaccess

Laravel liefert eine public/.htaccess mit. Stelle sicher, dass mod_rewrite aktiv ist.

Virtual Host (Beispiel)

<VirtualHost *:80>
    ServerName deine-domain.de
    DocumentRoot /var/www/zerrocms/public

    <Directory /var/www/zerrocms/public>
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>

Document Root muss immer .../public sein. Nach Änderung: sudo systemctl reload apache2 (bzw. httpd).

🟢 Nginx

Nginx hat keine .htaccess; die Weiterleitung muss in der Server-Konfiguration stehen.

Server-Block (Beispiel)

server {
    listen 80;
    server_name deine-domain.de;
    root /var/www/zerrocms/public;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    index index.php;

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_hide_header X-Powered-By;
    }
}

root muss auf .../public zeigen. Nach Änderung: sudo nginx -t && sudo systemctl reload nginx.

🪟 IIS (Windows Server)

Für IIS wird ein web.config in public/ benötigt (Laravel bringt oft eine Vorlage mit).

Wichtig

  • Physischer Pfad der Website/Application = Ordner public deines Projekts.
  • URL Rewrite Module für IIS installieren (Microsoft oder Hosting-Anbieter).
  • PHP über FastCGI anbinden und SCRIPT_FILENAME korrekt setzen.

Inhalt von public/web.config (Ausschnitt): Rewrite-Regeln für Laravel (alle Anfragen auf index.php leiten). Siehe Laravel-Dokumentation für die aktuelle web.config-Vorlage.

🔷 Plesk Panel

  1. Document Root: Domains → deine Domain → Hosting & DNS → Hosting-Einstellungen → Dokumentenstamm auf .../public setzen (z. B. /var/www/vhosts/deine-domain.de/httpdocs/public).
  2. PHP: Domains → PHP-Einstellungen → PHP 8.2 oder 8.3, Extensions: mbstring, openssl, pdo_mysql, tokenizer, xml, ctype, json, bcmath, fileinfo.
  3. Dateien: Projekt per Git/FTP/File Manager in z. B. httpdocs legen; Document Root zeigt auf httpdocs/public.
  4. Datenbank: Databases → Add Database anlegen, Zugangsdaten notieren.
  5. .env: Im Projekt-Root anlegen (aus .env.example), DB-Daten, APP_URL, APP_KEY eintragen.
  6. SSH/Terminal: composer install --no-dev --optimize-autoloader, php artisan migrate --force, php artisan storage:link, php artisan config:cache.
  7. Admin: User per Tinker anlegen oder unter /admin registrieren (falls aktiviert).
  8. Rechte: storage/ und bootstrap/cache/ für Webserver-Benutzer beschreibbar (z. B. 775).

📦 cPanel

Document Root auf public setzen (Domains → Domains → Document Root auf public zeigen, z. B. public_html/public oder Projekt in Unterordner und Document Root dorthin).

  • PHP: MultiPHP oder Select PHP Version → 8.2+, benötigte Extensions aktivieren.
  • Datenbank: MySQL Database Wizard nutzen, User zuweisen.
  • .env: Im Projekt-Root (nicht in public) anlegen und anpassen.
  • Terminal/SSH: composer install --no-dev --optimize-autoloader, php artisan migrate --force, php artisan storage:link, php artisan config:cache.

Falls kein SSH: Einmal-Skripte über „Run script“ oder PHP-CLI-Cron des Hosters ausführen.

🦬 Caddy

Minimaler Server-Block mit PHP-FPM:

deine-domain.de {
    root * /var/www/zerrocms/public
    encode gzip
    file_server
    php_fastcgi unix//var/run/php/php8.2-fpm.sock
    try_files {path} {path}/ /index.php?{query}
}

root zeigt auf public. Caddy neu laden nach Änderung der Caddyfile.

⚙️ .env & Artisan-Befehle

Für alle Server: Im Projekt-Root .env anlegen (z. B. aus .env.example). Mindestens:

APP_NAME=ZerroCMS
APP_ENV=production
APP_DEBUG=false
APP_URL=https://deine-domain.de

DB_CONNECTION=mysql
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=dein_db_name
DB_USERNAME=dein_db_user
DB_PASSWORD=dein_db_passwort

QUEUE_CONNECTION=sync
SESSION_DRIVER=database
CACHE_STORE=database

CLAN_NAME="Dein Clan"
CLAN_THEME=default
APPLICATION_NOTIFY_EMAIL=admin@deine-domain.de

Befehle (einmalig bzw. nach Deploy):

php artisan key:generate   # falls APP_KEY leer
composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan storage:link
php artisan config:cache
php artisan view:cache
